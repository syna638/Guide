Status update notifications for users.

ğŸ”„ Data Flow: Submitting a Complaint

Here's what happens when a citizen submits a new complaint:

User Fills Form

Citizen fills out the ComplaintForm component with title, description, category, location, and uploads a photo.

Image Uploads to Supabase Storage

The photo is uploaded to the complaint-images bucket in Supabase Storage. A public URL is returned.

Server Action Called

The form calls a Server Action (createComplaint) in lib/actions/complaints.ts.

Authentication Verified

The action uses auth() to get the current user's Clerk ID.

Data Inserted into Database

Supabase client inserts a new row into the complaints table with all the data.

UI Updates

The complaint list revalidates and shows the new complaint. User is redirected to the complaints page.
