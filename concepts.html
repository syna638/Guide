What is it?

The Frontend is everything you see and interact with on a website. It's the visual part ‚Äì buttons, forms, images, text, colors, and animations.

Real-World Analogy

üè™ Think of a Restaurant

The frontend is like the dining area of a restaurant. It's where customers sit, look at the menu, and place orders. The waiter (your web browser) shows you the menu (the webpage) and takes your order (your actions like clicking buttons).

What is it?

The Backend is the hidden machinery behind a website. It processes requests, runs business logic, talks to databases, and sends data back to the frontend.

Real-World Analogy

üç≥ Think of a Restaurant

The backend is like the kitchen. Customers don't see it, but that's where all the magic happens. When you place an order (submit a complaint), the kitchen (backend) prepares your food (processes your data) and sends it back out (returns a response).

In Smart Civic Connect

The backend handles:

  • Saving complaints to the database
  • Uploading images to cloud storage
  • Verifying that you're logged in
  • Sending notification emails
  • Processing admin actions

üîå API (Application Programming Interface)

What is it?

An API is a way for different software to communicate with each other. It's like a messenger that carries requests from the frontend to the backend and brings back responses.

Real-World Analogy

üçΩÔ∏è Think of a Restaurant

The API is like the waiter. You tell the waiter what you want (make a request), the waiter goes to the kitchen (backend), and brings back your food (response). You never go into the kitchen yourself ‚Äì the waiter is the intermediary.

Types of API Requests

Method Purpose Example
GET Retrieve data Get list of complaints
POST Create new data Submit a new complaint
PUT/PATCH Update existing data Update complaint status
DELETE Remove data Delete a draft complaint

In Smart Civic Connect

When you submit a complaint, the frontend sends a POST request to the API endpoint /api/complaints. The backend processes it and returns a success or error response.

üóÑÔ∏è Database

What is it?

A Database is a structured collection of data that's stored electronically. Think of it as organized digital filing cabinets where all your application's information lives.

Real-World Analogy

üìÅ Think of a Library

A database is like a library's catalog system. Every book (data) has a specific place, is organized by categories, and can be quickly found using the catalog (database queries). You can add new books, find existing ones, update information, or remove outdated entries.

Types of Databases

  • SQL Databases (PostgreSQL, MySQL): Data is stored in tables with rows and columns, like spreadsheets. Good for structured data.
  • NoSQL Databases (MongoDB): Data is stored in flexible documents. Good for unstructured data.

In Smart Civic Connect

We use Supabase, which is powered by PostgreSQL (a SQL database). Our tables include:

  • complaints - Stores all submitted complaints
  • categories - Types of complaints (Roads, Water, etc.)
  • profiles - User information linked to authentication
  • notifications - Status update alerts for users

üõ°Ô∏è Middleware

What is it?

Middleware is software that sits between the frontend and backend, checking every request before it reaches its destination. It's like a security checkpoint.

Real-World Analogy

üè¢ Think of a Building

Middleware is like building security. Before you can enter certain floors (protected pages), security checks your ID (authentication). Different people have access to different areas based on their credentials (authorization).

What Middleware Does

  • Authentication Check: Are you logged in?
  • Authorization Check: Do you have permission to access this?
  • Request Modification: Add headers or transform data
  • Logging: Record what's happening for debugging
  • Redirects: Send users to different pages based on conditions

In Smart Civic Connect

Our middleware file (middleware.ts) handles:

  • Clerk Authentication: Checks if citizens are logged in for protected routes like /home and /complaints
  • Admin Authentication: Uses custom JWT to verify admin access for /admin/* routes
  • Session Refresh: Keeps database sessions active
  • Public Routes: Allows access to /, /sign-in, /sign-up without login

üîê Authentication & Authorization

What's the Difference?

ü™™

Authentication

"Who are you?" ‚Äì Proving your identity (logging in with username/password)

üé´

Authorization

"What can you do?" ‚Äì Checking your permissions (can you access admin pages?)

Real-World Analogy

üé™ Think of a Concert

Authentication is showing your ID to prove you bought a ticket. Authorization is whether your ticket gives you access to VIP areas or just general admission.

In Smart Civic Connect

We have TWO separate authentication systems:

1. Clerk (for Citizens)

  • Handles sign up and sign in
  • Supports email, Google, and social login
  • Manages user sessions automatically
  • Protects citizen routes: /home, /complaints

2. Custom JWT (for Administrators)

  • Separate login at /admin/login
  • Uses secure JWT tokens stored in cookies
  • Credentials stored securely (hashed password)
  • Protects admin routes: /admin/dashboard, etc.
‚ö†Ô∏è Why Two Systems?

Keeping admin auth separate from citizen auth adds an extra layer of security. Even if someone gets access to a citizen account, they still can't access admin features without admin credentials.

üéüÔ∏è JWT (JSON Web Token)

What is it?

A JWT is a secure way to transmit information between parties. It's like a digitally signed ticket that proves you're authenticated.

Structure of a JWT

A JWT has three parts separated by dots:

xxxxx.yyyyy.zzzzz
Header.Payload.Signature
  • Header: Algorithm used for signing
  • Payload: The actual data (user ID, expiration time)
  • Signature: Cryptographic proof that the token hasn't been tampered with

Real-World Analogy

üé´ Think of a Wristband

At a music festival, you get a wristband. Security can quickly see you've paid (the wristband is your JWT). It's hard to fake (cryptographically signed), and it might expire at the end of the day (token expiration).

In Smart Civic Connect

Admin sessions use JWT stored in an admin-session cookie. When an admin logs in:

  1. Server verifies the password
  2. Creates a JWT with admin info and expiration
  3. Stores the JWT in a secure cookie
  4. Middleware reads this cookie on every admin request
  5. If valid, access is granted; if expired, redirect to login

üîë Clerk

What is it?

Clerk is a third-party authentication service that handles all the complex parts of user management: signup, login, password reset, email verification, and social login (Google, Facebook, etc.).

Why Use Clerk?

  • Security: Authentication is hard to get right. Clerk handles security vulnerabilities for you.
  • Speed: Adds full auth in minutes instead of weeks of development.
  • Features: Social login, multi-factor auth, email verification built-in.
  • Compliance: Handles privacy regulations like GDPR.

Real-World Analogy

üè® Think of a Hotel Key Card System

Instead of making your own keycard system, hotels buy one from a specialized company. It's secure, tested, and works reliably. Clerk is like that specialized keycard company for websites.

Key Clerk Components in SCC

  • ClerkProvider - Wraps the entire app to provide auth context
  • SignIn / SignUp - Ready-made login and registration components
  • UserButton - Profile dropdown with sign out option
  • auth() - Server function to get current user
  • clerkMiddleware - Protects routes server-side

üì± PWA (Progressive Web App)

What is it?

A PWA is a website that can be installed on your device and used like a native app. It combines the reach of the web with the experience of an app.

PWA Features

  • Installable: Add to home screen on phones and desktops
  • Offline Support: Works without internet (for cached pages)
  • Push Notifications: Can send notifications like native apps
  • Fast: Loads quickly, even on slow networks
  • No App Store: No need to download from Play Store or App Store

Key Files

  • manifest.json - Tells the browser how to install the app
  • Service Worker - Background script that enables offline features
  • App Icons - Various sizes for different devices

üì≤ TWA (Trusted Web Activity)

What is it?

A TWA wraps your PWA in an Android app container so you can publish it on the Google Play Store. It's essentially your website running in full-screen mode as an Android app.

Benefits

  • Publish your web app on Google Play Store
  • Users get the familiar "Install from Play Store" experience
  • Auto-updates as your website updates (no app review needed)
  • Access to some native Android features
üìò Note

TWA requires your PWA to pass certain quality checks (Lighthouse score, proper manifest, HTTPS, etc.).

üìù TypeScript

What is it?

TypeScript is JavaScript with added "types." Types help catch errors before running the code, making development safer and more predictable.

Example

// JavaScript - no error prevention
function add(a, b) {
  return a + b;
}
add("5", 3); // Returns "53" (string concatenation - probably a bug!)

// TypeScript - catches errors
function add(a: number, b: number): number {
  return a + b;
}
add("5", 3); // ERROR! "5" is not a number

Why Use TypeScript?

  • Catch errors early - Before the code even runs
  • Better autocomplete - IDE knows what properties/methods are available
  • Self-documenting - Types explain what data looks like
  • Refactoring confidence - Change code knowing errors will be caught
